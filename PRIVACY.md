# 아이온2 레기온 봇 개인정보처리방침

종류, 목적, 보유 기간, 이용자의 권리를 규정합니다. 본 방침은 대한민국 「개인정보 보호법」
및 관련 법령을 준수합니다.

## 2. 수집·처리하는 정보

### 2.1 Discord 가 자동 제공하는 식별자
| 항목 | 설명 |
|---|---|
| Discord 서버 ID (`guild_id`) | 명령어가 실행된 디스코드 서버의 식별자 |
| Discord 사용자 ID (`user_id`) | 명령어를 실행한 사용자의 식별자 |
| Discord 채널 ID / 음성 채널 ID | 알림이 발송될 채널 |
| Discord 역할 ID | 알림 시 멘션할 역할 (선택) |

### 2.2 이용자가 직접 입력한 내용
| 명령어 | 저장 항목 |
|---|---|
| `/빌드등록` | 직업명, 빌드명, 설명, 스탯, 스킬 텍스트 |
| `/일정등록` | 일정 제목, 일시, 설명, 알림 옵션 |
| `/장비생성` 이후 강화/돌파 흐름 | 장비 이름, 종류, 강화/돌파 등급, 누적 사용량 |

### 2.3 운영 로그
명령어 실행 사실(명령어명, 디스코드 사용자 태그, 서버 ID, 실행 시각)이 운영 서버의
컨테이너 로그에 기록되며 Docker 로그 로테이션 정책(최대 약 50MB, 5개 파일)에 따라
일정 기간 후 자동 폐기됩니다.

## 3. 수집하지 않는 정보
- 이메일 주소, 실명, 전화번호 등 디스코드 외부의 실제 신원 정보
- IP 주소 (별도 로깅하지 않음)
- 디스코드 메시지 본문 (`MESSAGE CONTENT INTENT` 미사용)
- 다이렉트 메시지(DM)
- 사용자 음성 데이터 (TTS 는 봇이 발화만 하며, 사용자 음성을 녹음·수신하지 않음)
- 결제 정보 (봇은 무료이며 결제 기능이 없음)

---

## 4. 처리 목적
- 빌드 / 일정 / 장비 데이터의 저장 및 조회 기능 제공
- 일정 알림 발송 (지정 채널 메시지, 역할 멘션, 음성 채널 TTS)
- 본인 소유권 확인 (예: 본인이 등록한 빌드만 삭제 가능)
- 서버별 데이터 격리 (다른 디스코드 서버로 누출 방지)

---

## 5. 보유 기간
- **이용자가 직접 입력한 데이터**: 이용자 본인 또는 서버 관리자가 삭제 명령을 실행할 때까지 보관
- **봇이 서버에서 추방·차단된 경우**: 운영자에게 요청 시 합리적 기간 내에 해당 서버 데이터 일괄 삭제
- **운영 로그**: Docker 로그 로테이션 기준 자동 폐기 (보통 수일 ~ 수십일)
- **데이터베이스 백업본**: 운영자 서버에서 최대 14일 보관 후 자동 삭제

---

## 6. 제3자 제공·결합 서비스
봇은 수집한 정보를 제3자에게 판매·임대하지 않습니다. 다만 봇이 정상 동작하기 위해
아래 외부 서비스와 결합됩니다.

| 서비스 | 처리되는 항목 | 목적 |
|---|---|---|
| **Discord** (Discord Inc.) | 모든 명령어 입출력, 메시지, 음성 트래픽 | 봇 운영 플랫폼 |
| **Google Translate TTS** (`translate.google.com`) | 알림 문구 텍스트 (예: "공성 일정이 10분 후 시작됩니다") | 음성 합성. **사용자 ID·서버 ID 등 식별자는 전송되지 않음** |

Discord 의 개인정보 처리에 대해서는 https://discord.com/privacy 를 참조하십시오.

## 7. 이용자 권리
이용자는 다음 권리를 직접 행사할 수 있습니다.

| 권리 | 방법 |
|---|---|
| 본인 데이터 **열람** | `/빌드목록`, `/일정목록`, `/장비목록` 명령 실행 |
| 본인 데이터 **삭제** | `/빌드삭제`, `/일정삭제`, `/장비삭제` 명령 실행 |
| 서버 전체 데이터 삭제 | 서버 관리자가 봇을 서버에서 추방 후 본 방침 1항 문의처로 요청 |
| 본인의 모든 데이터 삭제 | 본 방침 1항 문의처에 Discord 사용자 ID 와 함께 요청 |

요청 접수 후 합리적인 기간(통상 14일) 이내에 처리합니다.

## 8. 보안 조치
- 봇 데이터는 운영자의 서버 내부 디스크에만 저장되며, 외부 네트워크에 공개되지 않습니다.
- Discord 봇 토큰 등 인증 정보는 환경변수로만 관리되며 소스코드에 포함되지 않습니다.
- 데이터베이스 파일은 정기적으로 백업되며, 백업본은 운영자만 접근할 수 있습니다.
- 봇이 동작하는 서버는 운영체제 보안 업데이트가 적용됩니다.
